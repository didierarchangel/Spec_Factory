# 🗄️ Configuration des Bases de Données - Backend

Ce dossier contient **3 configurations d'environnement** pour le backend, chacune avec une base de données différente.

---

## 📋 Configurations Disponibles

### 1️⃣ **MongoDB** (`.env.mongodb`)
**Type**: NoSQL Document-based  
**Environnement**: Développement / Production  
**Port par défaut**: 27017

```bash
cp .env.mongodb .env
```

**Caractéristiques**:
- ✅ Flexible et scalable
- ✅ Idéal pour les données non-structurées
- ✅ Supporte MongoDB Atlas (cloud) et local
- ✅ Excellente performance pour les requêtes complexes

**Installation locale**:
```bash
# Windows (via Chocolatey)
choco install mongodb

# macOS (via Homebrew)
brew tap mongodb/brew
brew install mongodb-community

# Docker
docker run -d -p 27017:27017 --name mongodb mongo
```

**Connexion**:
```bash
# Local
mongodb://localhost:27017/drugstoredb

# Atlas (Cloud)
mongodb+srv://username:password@cluster.mongodb.net/drugstoredb
```

---

### 2️⃣ **PostgreSQL Local** (`.env.postgresql`)
**Type**: SQL Relational  
**Environnement**: Développement  
**Port par défaut**: 5432

```bash
cp .env.postgresql .env
```

**Caractéristiques**:
- ✅ Puissant et fiable
- ✅ Excellentes performances SQL
- ✅ Support complet des transactions ACID
- ✅ Idéal pour les données structurées
- ⚠️ Installation locale requise

**Installation locale**:
```bash
# Windows (via installer)
# https://www.postgresql.org/download/windows/

# macOS (via Homebrew)
brew install postgresql@15

# Docker
docker run -d -p 5432:5432 \
  -e POSTGRES_PASSWORD=PASSWORD \
  -e POSTGRES_DB=drugstoredb \
  --name postgres postgres:15

# Linux (Ubuntu/Debian)
sudo apt-get install postgresql postgresql-contrib
```

**Configuration PostgreSQL Local**:
```bash
# Créer la base de données
createdb -U postgres drugstoredb

# Vérifier la connexion
psql -U postgres -h localhost -d drugstoredb

# Accéder à psql
psql -U postgres

# Définir le mot de passe
\password postgres
```

**Connexion**:
```
postgresql://postgres:PASSWORD@localhost:5432/drugstoredb
```

**Migration depuis MongoDB**:
```bash
# Utiliser un outil comme:
# - pgloader (recommended)
# - Mongoose to Typeorm/Sequelize migration
# - Script personnalisé
```

### 3️⃣ **Supabase** (`.env.supabase`)
**Type**: PostgreSQL Managé sur le Cloud  
**Environnement**: Production / Staging  
**Provider**: Supabase (Firebase alternative)

```bash
cp .env.supabase .env
```

**Caractéristiques**:
- ✅ PostgreSQL managé et sécurisé
- ✅ Inclus: Base de données, Auth, Realtime, Storage
- ✅ Excellent pour la scalabilité
- ✅ Gratuit jusqu'à certaines limites
- ✅ Idéal pour la production

**Configuration Supabase**:

1. **Créer un compte**:
   - Aller sur https://app.supabase.com
   - S'enregistrer avec GitHub ou Email

2. **Créer un projet**:
   - Click "New Project"
   - Choisir la région (ex: US East)
   - Définir un mot de passe strong
   - Attendre l'initialisation (~2 min)

3. **Récupérer les credentials**:
   - Project Settings → Database
   - Copier `Project ID`
   - Copier `Database Password`
   - API Keys section pour les clés API

4. **Remplir le `.env.supabase`**:
   ```env
   DATABASE_URL=postgresql://postgres.abc123def456:PASSWORD@db.abc123def456.supabase.co:5432/postgres
   SUPABASE_PROJECT_ID=abc123def456
   SUPABASE_API_KEY=eyJhbGc...
   SUPABASE_URL=https://abc123def456.supabase.co
   ```

**Connexion**:
```
postgresql://postgres.PROJECT_ID:PASSWORD@db.PROJECT_ID.supabase.co:5432/postgres
```

## 🚀 Comment Utiliser

### Basculer entre les configurations

```bash
# Pour MongoDB
cp .env.mongodb .env

# Pour PostgreSQL Local
cp .env.postgresql .env

# Pour Supabase
cp .env.supabase .env
```

### Démarrer le serveur

```bash
# Installer les dépendances
npm install

# Démarrer en développement
npm run dev

# Ou avec la configuration spécifique
NODE_ENV=development npm run dev
```

### Vérifier la connexion

```bash
# MongoDB
npm run db:migrate:mongodb

# PostgreSQL
npm run db:migrate:postgresql

# Supabase
npm run db:migrate:supabase
```

## 💡 Recommandations

### Pour le Développement Local
👉 Utiliser **PostgreSQL Local** ou **MongoDB Local**
- Installation unique
- Pas d'accès internet requis
- Idéal pour les tests

### Pour la Production
👉 Utiliser **Supabase** (meilleure option)
- Sécurité renforcée
- Backups automatiques
- Support technique
- Scalabilité incluse

### Migration entre configurations
Si vous devez passer de MongoDB à PostgreSQL/Supabase:
```bash
# 1. Exporter depuis MongoDB
mongoexport --collection users --out users.json

# 2. Transformer en SQL
# Scripts custom ou outils comme pgloader

# 3. Importer dans PostgreSQL
psql -U postgres -d drugstoredb -f import.sql
```

## ❓ Dépannage

### Erreur: "Cannot connect to database"
```bash
# Côté MongoDB
# 1. Vérifier que MongoDB est en cours d'exécution
mongod --version

# 2. Vérifier le port 27017
netstat -an | findstr 27017

# Côté PostgreSQL
# 1. Vérifier la connexion
psql -U postgres -h localhost -d drugstoredb

# 2. Vérifier le mot de passe
# Réinitialiser dans psql: \password postgres
```

### Erreur: "Invalid DATABASE_URL format"
```bash
# Format MongoDB valide
mongodb://username:password@localhost:27017/database

# Format PostgreSQL valide
postgresql://username:password@host:port/database

# Format Supabase valide
postgresql://postgres.PROJECT_ID:PASSWORD@db.PROJECT_ID.supabase.co:5432/postgres
```
