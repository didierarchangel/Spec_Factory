## [ ] 02_user_authentication_module
- [ ] **Initial Database Setup & Dependencies:**
    - [ ] Choose and configure the database connection (e.g., MongoDB with Mongoose).
    - [ ] Install core dependencies: `express`, `dotenv`, `bcrypt`, `jsonwebtoken`.
    - [ ] Install database driver/ORM (e.g., `mongoose`).
- [ ] **Configuration Management:**
    - [ ] Create `config/` directory and `config/db.js` for database connection details.
    - [ ] Create `config/jwt.js` for JWT secret and options (expiration).
    - [ ] Set up `.env` file for environment variables (DB_URI, JWT_SECRET).
- [ ] **User Data Model Implementation:**
    - [ ] Create `src/models/User.js` with schema fields: `_id`, `email`, `password`, `firstName`, `lastName`, `createdAt`, `updatedAt`.
    - [ ] Implement pre-save hook in `User.js` to hash `password` using `bcrypt` before saving.
- [ ] **Authentication Service & Controller:**
    - [ ] Create `src/services/auth.service.js` to encapsulate business logic for user registration and login.
    - [ ] Implement `registerUser` function in `auth.service.js`: validate input, check email uniqueness, hash password, save user.
    - [ ] Implement `loginUser` function in `auth.service.js`: find user by email, compare password, generate JWT.
    - [ ] Create `src/controllers/auth.controller.js` to handle request parsing and response formatting, calling `auth.service.js`.
- [ ] **Authentication Routes Definition:**
    - [ ] Create `src/routes/auth.routes.js` to define API endpoints.
    - [ ] Define `POST /api/auth/register` route, linking to `auth.controller.register`.
    - [ ] Define `POST /api/auth/login` route, linking to `auth.controller.login`.
    - [ ] Integrate `auth.routes.js` into the main `app.js` or `server.js` file.
- [ ] **JWT Generation and Validation:**
    - [ ] Implement a utility function (e.g., in `src/utils/jwt.js`) for generating signed JWTs upon successful login/registration.
    - [ ] Create `src/middleware/auth.middleware.js` to validate JWTs on protected routes.
    - [ ] Implement logic in middleware to extract token from `Authorization` header, verify its signature and expiration, and attach decoded user info to `req.user`.
- [ ] **Security Enhancements for Authentication:**
    - [ ] Implement input validation middleware (e.g., using `express-validator`) for `register` and `login` request bodies.
    - [ ] Implement rate limiting middleware (e.g., `express-rate-limit`) for `POST /api/auth/register` and `POST /api/auth/login` to prevent brute-force attacks.
    - [ ] Ensure all sensitive data (passwords, JWT secrets) are handled securely and not exposed in logs or responses.
